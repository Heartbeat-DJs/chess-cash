## Key Decisions Made

### Autonomous Worker Pipeline (for DarkMatterNucleus project)
1. **Architecture decided:** Claude/Antigravity acts as orchestrator + quality gate, dispatching grunt work to a local/cloud LLM running in sandboxed Docker containers
2. **Model selection:** MiniMax M2.7 via API (released March 18, 2026)
   - 230B MoE, 10B active params, 204k context, 131k max output
   - SWE-Bench Pro: 56.22%, drastically reduced hallucinations vs M2.5
   - 20% more token-efficient than M2.5
   - Pricing: $0.30/M input, $1.20/M output (~$0.04/task)
   - **Proprietary API only** — no Ollama cloud mode, needs direct API integration
3. **Fallback model:** Qwen3.5-27B on user's RTX 3090 (24GB VRAM) for zero-cost simple boilerplate tasks
4. **Sandbox model:** All worker output goes to isolated Docker containers on worker branches → Claude reviews diffs → approve/reject with feedback loop (max 3 iterations)
5. **Infrastructure:** Leverages existing n8n automation for task queue + dispatching
6. **Multi-agent:** 3-4 parallel Docker containers can run simultaneously, each on its own branch

### MiniMax M2.5 vs M2.7 Comparison
- M2.7 chosen over M2.5 primarily for:
  - Hallucination reduction (score went from -40 to +1)
  - 20% token efficiency
  - Self-evolving training architecture
- M2.5 had slightly higher SWE-Bench Verified (80.2% vs 78%) but M2.7 benchmarks on harder tests
